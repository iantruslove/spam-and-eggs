API

Status: work in progress

The point of this tool was to provide a way to generate a whole number of emails representing an individual's or an organization's email interactions over a period of time.

This website provides examples of what the API can provide, but the API is actually way better for generating "real" test corpuses.

Right now, I don't have any API docs for you. Sorry, I'm working on it. If you're really keen, check out the JS source, you'll get the idea.

Models

The personal information is generated from a couple of lists of common names and email domains, plus some basic frequency distribution models. I use the generators in test.check to select items from lists, because it makes it really easy.

Using demographic data to get common first and last name frequencies, it's possible to generate eerily familiar names and email addresses. If you know anybody generated by this site list, I promise it's just a coincidence...

The email text is generated with a Markov chain. Right now the transition probabilities between states were generated by me by hand, which explains the utter ridiculousness, simplisticness and lack of variation in the text.

Tech

Spam and Eggs is written in Clojure, and hosted by Heroku . Both of these are quite great.

Inspiration

I spend my work time working with systems that analyze emails. The quality of results of the statistical, text analytics and machine learning work is quite dependent on the input data, and I was getting tired of hacking emails from the Enron corpus into the shape I wanted. At first I was just creating email addresses with real-looking names, but it quickly spread into text generation using Markov. It's a fun toy.
